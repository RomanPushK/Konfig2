# Konfig2 — Анализатор зависимостей APT-пакетов
## 1. Общее описание

Konfig2 — это консольное Java-приложение, предназначенное для загрузки и анализа данных о пакетах из APT-репозитория, последующего построения дерева зависимостей и вывода его в удобном древовидном текстовом представлении.

Программа способна работать как с локальным файлом (--test режим), так и с реальным удалённым репозиторием через URL.
Поддерживаются фильтры для исключения зависимостей и предотвращение зацикливания при построении графа.

## 2. Описание всех функций, классов и параметров
   Классы, использованные в программе
   Main	Главный класс программы, содержащий логику ввода, обработки, загрузки и вывода результатов
   Config	Внутренний класс для хранения параметров конфигурации, полученных из аргументов командной строки
   AptPackage	Представление одного APT-пакета (имя + список зависимостей)
   Методы
   main	Точка входа программы: получение настроек, загрузка пакетов, построение карты зависимостей и вывод дерева
   parseArgs	Обрабатывает аргументы командной строки (--package, --repo, --test, --filter)
   loadLocal	Загружает данные о пакетах из локального файла в текстовом формате
   loadRemote	Загружает данные по HTTP(S) из Packages-файла удалённого APT-репозитория
   parsePackages	Анализирует исходный текст Packages и собирает список объектов AptPackage
   parseDependencies	Обрабатывает строку зависимостей: удаляет версии, разделяет альтернативы и формирует чистый список
   buildDependencyGraphBFS	Построение графа зависимостей с помощью обхода в ширину (BFS)
   printGraphAsTree	Рекурсивный вывод дерева зависимостей с визуальными отступами
   printTree	Альтернативный метод отображения зависимостей с проверкой циклов
   printIndent	Вспомогательная функция для красивого форматирования отступов
   Аргументы командной строки
   Аргумент	Описание
   --package <name>	Имя анализируемого пакета (корневой элемент дерева)
   --repo <file_or_url>	Путь к локальному файлу или URL APT-репозитория
   --test	Включает режим чтения из локального файла, а не из интернета
   --filter <substring>	Исключает из анализа пакеты, содержащие указанную подстроку
   --help	Выводит справку по параметрам
## 3. Команды для сборки и запуска
   Компиляция
   ```javac Konfig2/Main.java```

Запуск с удалённым репозиторием
```java Konfig2.Main --package nginx --repo http://archive.ubuntu.com/ubuntu/dists/jammy/main/binary-amd64```

Запуск на локальном файле Packages
```java Konfig2.Main --package A --repo testrepo.txt --test```

Использование фильтра
```java Konfig2.Main --package A --repo testrepo.txt --test --filter dev```

## 4. Пример вывода

Входной файл содержит зависимости:

````
A -> B, C, F
B -> D
F -> K
````

Запуск:

java Konfig2.Main --package A --repo testrepo.txt --test


Вывод:
````
A
--- B
    --- D
--- C
--- F
--- K
````
